# Database Migrations

This directory contains all database migration files for the POS application.

## File Naming Convention

Migrations should be named with the following format:
```
YYYYMMDDHHMMSS-DescriptiveName.sql
```

Examples:
- `20250115000000-CreateStationPricelistTable.sql`
- `20250115000001-FixStationStatusValues.sql`

## Migration Types

1. **TypeORM Migrations** (`.cjs` files)
   - Generated by TypeORM CLI
   - Used for schema changes managed by TypeORM
   - Examples: `1700000000001-AddStationIdToBill.cjs`

2. **Manual SQL Migrations** (`.sql` files)
   - Custom SQL scripts for data transformations
   - Used when TypeORM migrations are not sufficient
   - Examples: `20250115000001-FixStationStatusValues.sql`

## Running Migrations

```bash
# Run all migrations
npm run migration:run

# Run specific migration manually
mysql -u root -ppassword bizmaster < src/backend/config/migrations/20250115000001-FixStationStatusValues.sql
```

## Migration History

- `20250115000000-CreateStationPricelistTable.sql` - Created station_pricelist junction table for M:M relationship
- `20250115000001-FixStationStatusValues.sql` - Standardized station status values to active/inactive
- `20250115000002-UpdateDefaultStatusToInactive.sql` - Updated default status for new stations and pricelists to inactive
